## 学到的经验教训

### 1. 融合优于替换
学习他人经验时，保持核心身份认同。不是全盘复制，而是选择性移植 + 本土化改造。

### 2. 结构化的价值
Kimi Claw 的记忆系统分层清晰（daily/weekly/projects/people），值得借鉴。清晰的层级结构提升记忆检索效率。

### 3. 智能合并
功能重复的 skills 无需重复安装，差异化 skills 互补保留。例如：
- browser-tool → 融入 agent-browser
- frontend-aesthetics → 融入 ui-design
- infographic-prompt-generator → 融入 image-assistant

### 4. API 策略
需要额外 API/账号的 skills 带来维护负担，明智地跳过：
- nanobanana-infographic（需 nanobanana API）
- xiaohongshu-automation（需小红书账号）
- moltbook-reporter（需 Moltbook 账号）
- youmind（需 Youmind 账号）

### 5. 单一数据源原则
每类信息只记录在一个地方，避免分散和重复：
- TASKS.md：待办任务、问题追踪
- STATE.md：当前会话状态
- YYYY-MM-DD.md：每日原始日志
- MEMORY.md：长期记忆、经验教训
