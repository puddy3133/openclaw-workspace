- QMD 格式、简洁回复风格（持续接受，无修正）
- 技术调研用对比表格呈现（多次接受）

### 显式修正记录
> 每次用户修正输出后，按以下格式追加：
> `- [YYYY-MM-DD] 修正内容 | 原因 | 影响范围`

- [待记录 — 下次修正发生时自动追加到此处]

### 反馈捕获规则
当检测到以下信号时，自动记录到本节：
1. **显式修正**：用户说"不对"、"不是这样"、"改成X"、"别这样做" → 记录修正内容和原因
2. **显式肯定**：用户说"好的"、"完美"、"就这样"、且方法非显然 → 记录被验证的方法
3. **隐式修正**：用户重新编辑了 Agent 的输出（diff > 30%）→ 记录差异摘要
4. **偏好变化**：用户对同类任务给出不同于历史偏好的指令 → 更新偏好并标注变化日期
