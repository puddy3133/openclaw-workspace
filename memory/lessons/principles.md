# 工作原则与技术偏好

## 沟通偏好（用户：国栋）

- 简洁直接，可执行
- QMD 格式反馈（Quick/Measured/Deep，按场景选深度）
- 不喜欢废话开头（"好的！"、"没问题！"）
- 时间表达优先 Asia/Shanghai

## 配置变更协议

在修改任何配置文件前，必须：
1. 验证判断正确性（测试/验证）
2. 深度思考是否有不改配置的替代方案
3. 最优方案不涉及配置变更时，优先非配置方案
4. 改配置前充分测试（无运行时影响、无稳定性问题）
5. 全部通过后才执行修改

## 模型选择原则（见 SOUL.md 第三章）

- 默认：主模型 `primary` ,主模型断联的情况下，根据 `fallbacks` 顺序依次轮寻
- 空响应立即降级，不等待固定时间

## 技术栈

- 平台：OpenClaw
- 主模型：主模型 `primary` 
- 渠道：飞书（小乔）
- 时区：Asia/Shanghai
- 向量记忆：memory-tfidf-plugin + baai/bge-m3
