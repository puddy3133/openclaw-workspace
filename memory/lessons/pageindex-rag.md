# 无向量RAG系统（PageIndex）

**学习日期**: 2026-04-07  
**来源**: https://waytoagi.feishu.cn/wiki/Fj5UwmoRKiEP9mkRL3ccCEPvnLg

## 核心概念

PageIndex是一种**无需向量、基于推理的RAG系统**。它将文档转化为树形结构，由大语言模型（LLM）通过推理遍历该树形结构以获取答案，全程无需嵌入向量，也无需相似度检索。

这与人类查找信息的方式相似：先看目录，找到章节，查看小节，再定位到具体内容。

## 工作原理

### 步骤1：文档解析为层次化树形结构
- 将文档发送给LLM，拆分为顶级章节
- 对于长章节，继续拆分为子章节
- 生成多级树结构：短章节为叶子节点，长章节为内部节点

### 步骤2：自下而上总结每个节点
- 从叶子节点到根节点遍历树
- 每个叶子节点由LLM生成简短摘要
- 内部节点由子节点摘要构建摘要
- 根节点最终得到整个文档的摘要

### 步骤3：保存索引
- 将构建好的树形索引持久化存储

### 步骤4：树搜索检索
- 用户提问时，LLM从根节点开始推理
- 逐层判断哪个分支可能包含答案
- 最终定位到最相关的叶子节点

## 可复用经验

1. **用树形结构替代平铺的向量检索** - 适合有明确层级结构的文档
2. **利用LLM的推理能力进行层次化决策** - 推理过程可解释，可追踪答案来源
3. **模仿人类认知方式设计AI系统** - 目录→章节→小节→内容的查找方式

## 潜在应用场景

- 企业内部知识库问答
- 技术文档检索
- 教材/论文内容定位
- 任何有明确层级结构的文档集合
