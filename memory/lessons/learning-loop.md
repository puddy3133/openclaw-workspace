# 学习闭环规则（Learning Loop）

> 基于 Hermes Agent 的学习机制，定义何时提取、如何提取、如何精炼推理模式

---

## 一、触发条件（何时提取）

满足以下**全部条件**时，触发学习闭环：

1. **任务成功完成**（用户确认或任务目标达成）
2. **方法非平凡**（非单步操作，包含推理、多步决策或新方法）
3. **用户未修改输出**（隐式接受）或**用户明确表示满意**

不触发的情况：
- 简单查询、单步命令执行
- 用户对输出有较大修改（此时应更新用户建模，而非提取 pattern）
- 与已有 pattern 完全重复

---

## 二、提取流程

```
任务完成
  ↓
自检：这个方法值得记录吗？
  ├── 否 → 跳过
  └── 是 → 搜索 patterns/ 中是否有类似 pattern
              ├── 有 → 对比：新方法更好吗？
              │         ├── 是 → 更新现有 pattern，记录改进原因
              │         └── 否 → 跳过
              └── 无 → 新建 pattern 文件
```

---

## 三、Pattern 文件格式

存入 `memory/patterns/{task-type}.md`（不带日期，同 task-type 更新原文件）：

```markdown
---
task-type: {类型，如 feishu-integration / tts-setup / multi-agent}
context: {触发场景一句话描述}
created: YYYY-MM-DD
updated: YYYY-MM-DD
version: 1
tags: [tag1, tag2]
refs: 0
---

## 场景
{什么情况下遇到这个问题}

## 方法
{具体推理步骤或操作链路，要足够具体可复现}

## 效果
{结果如何，用户反应}

## 注意事项
{坑点或边界条件}

## 更新记录
- v1 (YYYY-MM-DD): 初始创建
```

---

## 四、精炼规则

- **新方法持续优于旧 pattern** → 更新现有文件，在文件末尾追加 `## 更新记录` 说明改进点
- **用户修正了某个 pattern 的做法** → 立即更新，记录原因（"用户指出 X 方式更好，因为..."）
- **Pattern 超过 6 个月未被引用** → 移入 `patterns/archive/`

---

## 五、检索规则（任务开始时）

开始非平凡任务前：
1. 识别任务类型（task-type）和关键词（tags）
2. 搜索 `patterns/` 中匹配的文件
3. 找到匹配时：提示"上次类似任务（{date}）的做法是...，是否沿用？"
4. 用户确认沿用 → 直接应用；用户修改 → 记录差异

---

## 六、反馈信号捕获（Hermes 用户建模增强）

当检测到用户修正或肯定信号时，立即执行：

```
用户消息到达
  ↓
包含修正信号？（"不对"/"改成"/"错了"/"重新来"）
  ├── 是 → 提取修正内容 + 原因
  │         → 追加到 people/国栋.md「显式修正记录」
  │         → 检查是否关联某个 pattern → 是则更新 pattern 注意事项
  └── 否 → 包含肯定信号？（"完美"/"就这样"/"exactly"）
            ├── 是 + 方法非显然 → 追加到「隐式接受」
            └── 否 → 不处理
```

**记录格式**：`[YYYY-MM-DD] 修正/肯定内容 | 原因 | 影响范围`

---

## 七、种子 Patterns（初始化）

以下已有经验可作为初始 pattern 提取来源：
- `lessons/learned.md` 中的多步操作记录
- `lessons/rules.md` 中的操作流程
- 近期日志中的成功任务链路
