## 场景
当 `learning-queue` 中有待消化的链接，或用户发送长篇文档/代码库要求学习时。

## 方法
1. **三轮过滤法**：
   - 第一轮：`WebFetch` 抓取并生成摘要（确认是否值得存入长期记忆）。
   - 第二轮：结构化提取。如果是技术调研，强制输出为「对比表格 + 核心结论 + 可借鉴点」（遵循 `people/国栋.md` 偏好）。
   - 第三轮：映射。将提取的知识点分别存入 `lessons/` (事实知识) 或 `patterns/` (方法论)。
2. **索引挂载**：在 `INDEX.md` 或 `MEMORY.md` 中增加一行索引，标记为 `🆕` 或 `今日更新`。
3. **过期清理**：如果新知识覆盖了旧知识，立即更新旧文件并添加 `(已更新)` 标记。

## 效果
知识不再堆积在“收藏夹”，而是转化为 Agent 可检索的工具，提高了后续任务的执行效率。

## 注意事项
- 不要全量搬运文档，只记录“非直觉”和“差异化”的信息。
- 必须包含 `Source URL` 以备追溯。
