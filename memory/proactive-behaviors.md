# 主动行为配置

> 最后更新：2026-03-13 17:29 (Asia/Shanghai)

## 主动汇报类

### 每日下班总结
- **触发时间**：每天 18:00 (Asia/Shanghai)
- **内容**：今日工作回顾（1-2句）+ 明天待办
- **渠道**：飞书私聊
- **格式**：简洁、可执行

### 紧急事项即时提醒
- **触发条件**：
  - 紧急邮件到达
  - 日程变动（未来24小时内）
  - 重要截止日期临近
- **渠道**：飞书私聊
- **优先级**：高优先级立即推送，低优先级聚合后推送

## 学习闭环类（Hermes 机制）

### 任务完成后自检
- **触发条件**：非平凡任务成功完成（用户确认或目标达成）
- **执行步骤**：
  1. 判断：这个方法值得记录吗？（多步推理 + 成功 + 用户未大幅修改）
  2. 搜索 `patterns/` 中是否有类似 pattern
  3. 有则对比更新，无则新建 pattern 文件
  4. 若用户修改了输出，记录修正内容 → 更新 `people/国栋.md` 反馈信号
- **详细规则**：见 `lessons/learning-loop.md`

### 任务开始前检索
- **触发条件**：开始非平凡任务时
- **执行步骤**：
  1. 识别任务类型和关键词
  2. 搜索 `patterns/` 中匹配的历史模式
  3. 找到匹配时提示用户，询问是否沿用

### 启动时加载
- **触发条件**：每次会话启动（读取 INDEX.md 之后）
- **执行步骤**：
  1. 扫描 `patterns/` 目录，加载最近 5 个 pattern 摘要
  2. 读取 `people/国栋.md` 任务偏好，确认当前会话风格

## 背景运转类

### 记忆整理
- **频率**：每2-3天（Heartbeat 时执行）
- **内容**：
  - 扫描近期 `memory/day/YYYY-MM-DD.md`
  - 识别值得长期保留的内容
  - 更新到 MEMORY.md
- **原则**：只保留真正重要的，不凑数

### 待办检查
- **频率**：Heartbeat 时
- **内容**：
  - 检查 TASKS.md 中待办事项
  - 快到期的提前提醒
  - 已完成的标记清理

### Heartbeat 真实检查
- **不再只回 OK**
- **真实执行**：
  1. 定时任务遗漏检查
  2. 待办事项扫描
  3. 记忆整理（轮转）
  4. 邮件/日程检查（如有权限）
  5. 承诺事项跟进

## 探索类

### 话题追踪
- **触发**：用户提及某话题后，设为追踪项
- **行为**：过1-2天带着搜索结果回来
- **格式**：轻量级，不刷屏

### 彩蛋投放
- **频率**：每天最多1次
- **内容**：用户可能感兴趣的链接、想法、关联
- **原则**：有真实冲动才发，不为发而发

## 执行原则

1. **不打扰优先**：深夜（23:00-08:00）不发非紧急消息
2. **质量优先**：宁缺毋滥，不刷存在感
3. **聚合优先**：多项提醒合并发送，不打成碎片
4. **权限尊重**：邮件/日程等需授权后才检查
