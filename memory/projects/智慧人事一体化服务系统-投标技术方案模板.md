## 一、项目理解与需求分析

### 1.1 项目背景
深圳职业技术大学升格为职业本科后，业务场景更加丰富（本专科并存、跨区域办学），需建设智慧人事一体化服务系统，推动数据赋能，服务学校教育高质量发展。

### 1.2 建设目标
- 实现人事全生命周期管理（招聘→在职→离校）
- 打通数据孤岛，对接学校现有平台
- 提升人事管理效率与服务体验

### 1.3 核心需求理解
**技术要求核心：**
- B/S架构，JavaEE技术栈，关系型数据库
- 移动端HTML5，兼容iOS/安卓/鸿蒙
- 对接统一身份认证、一网通办、消息平台
- 等保二级测评，数据安全合规

**功能模块（13个）：**
角色化工作台 → 招聘进校 → 教工管理 → 薪酬福利 → 考核评价 → 职称岗聘 → 人才发展 → 教职工离校 → 教师数据质量管理 → 教师大数据分析 → 教师画像 → 人事移动 → 网络安全等级保护测评服务

## 二、总体技术架构

### 2.1 架构设计原则
- 高可用：支持3000并发，响应时间≤3秒
- 高安全：符合等保二级要求
- 高扩展：微服务架构，支持国产信创
- 高集成：统一认证、数据共享、服务集成

**核心理念：** 分阶段建设、迭代交付、持续优化

**实施路径：**
1. **第一阶段（初验）**：角色化工作台 + 招聘进校 + 教职工离校 + 教师画像（教学档案袋）+ 教工管理（人员信息/人员调配/减员业务）

### 3.7 安全保障体系与控制措施

#### 网络安全
- HTTPS加密传输
- 防SQL注入、XSS攻击
- 爬虫识别与拦截
- 日志审计（保留≥6个月）

#### 数据安全
- 敏感数据脱敏显示
- 分级权限管理
- 数据备份（本地+异地）
- 操作日志全记录

#### 应用安全
- 密码强度校验
- 弱口令禁止
- 恶意登录限制
- 明文禁止存储传输

### 3.8 AI应用与风险控制

#### AI应用场景
| 模块 | AI功能 | 价值 |
|------|--------|------|
| 招聘进校 | 简历智能解析、报名材料辅助审核 | 提升效率50% |
| 职称岗聘 | 申报资格智能判断 | 减少人工校验 |
| 年度考核 | 业绩数据辅助填写 | 数据一致性 |
| 大数据分析 | 师资趋势预测、AI人事日报 | 决策支持 |

#### 风险控制
- AI访问范围权限控制
- 人工复核机制
- 数据隐私保护
- 模型可解释性
