## 核心理念

### 1. Skill 设计哲学

**不是功能堆砌，而是方法论外化**

```
理解 → 判断 → 表达 → 传播
```

### 2. 红线思维（最重要）

**别只写"要什么"，要写"不要什么"**

以 `ljg-plain` 为例的9条红线：

| 红线 | 具体含义 |
|------|----------|
| **口语检验** | 读出声，像跟朋友说话 |
| **零术语** | 12岁孩子能复述 |
| **短词优先** | 「进行分析」→「看」 |
| **一句一事** | 长句拆短 |
| **具体** | 名词看得见，动词有力气 |
| **开头给理由** | 不铺垫、不背景 |
| **不填充** | 删开场白、拐杖词 |
| **信任读者** | 跳过软化、辩解 |
| **诚实** | 想不清楚就说想不清楚 |

### 3. 形式自由，下限锁死

- 不规定怎么写
- 但规定不能怎么写
- 由内容决定形式（类比、故事、问答、递进例子...）

## 13个 Skills 详解

### 分类一：认知原子（核心）

#### 1. ljg-plain - 白话引擎
**目标**: 把任何内容改写到聪明的12岁小孩也能懂
**核心**: 不是"简化"，而是"grok"（真正理解）
**红线**: 9条红线必须逐条过
**输出**: ~/Documents/notes/*.org

#### 2. ljg-word - 单词精通
**目标**: 深度拆解英语单词的核心语义
**方法**: 不是查词典，而是拆"词的骨头"
**三层次**: 原始画面 → 核心意象 → 为什么长成今天这个意思

#### 3. ljg-learn - 概念解剖
**目标**: 从8个方向切开一个概念
**八维度**: 历史、辩证、现象、语言、形式、存在、美感、元反思
**输出**: 压成一句顿悟

#### 4. ljg-rank - 降秩引擎
**目标**: 给一个领域，找出背后不可再少的独立生成器
**判据**: 能生成、最小化、相互独立、有预测力
**不是**: 总结关键点，而是找"牵引线"

#### 5. ljg-writes - 写作引擎
**目标**: 带着一个观点出发，在写的过程中想透
**姿态**: "一个人在想事情，碰巧被你看见"
**禁止**: 演讲腔、教学腔、"接下来我们讨论"

#### 6. ljg-invest - 投资分析
**目标**: 判断项目是否是「秩序创造机器」
**核心**: 不问值多少钱，问转不转得起来
**特点**: 最有判断框架的skill

### 分类二：阅读研究链

#### 7. ljg-paper - 论文阅读
**目标**: 为非学术人士提取论文核心想法
**重点**: 重理解不重批判
**输出**: 费曼翻译 + 洞见 + "对我有什么启发"

#### 8. ljg-paper-river - 论文溯源
**方法**: 倒读法，递归挖前序论文（最多5层）
**输出**: 从源头讲述问题演化史

### 分类三：传播可视化链

#### 9. ljg-card - 内容铸卡 ⭐核心
**目标**: 将内容转为PNG视觉卡片
**模式**: -l(长图) -i(信息图) -m(多卡) -v(视觉笔记) -c(漫画) -w(白板)
**依赖**: Playwright + Chromium
**视觉红线**: 禁Inter、禁纯黑、禁三等分、禁居中Hero、禁AI文案腔

#### 10. ljg-paper-flow - 论文→卡片
**链**: ljg-paper → ljg-card -c
**价值**: 读完论文直接做成可传播资产

#### 11. ljg-word-flow - 单词→卡片
**链**: ljg-word → ljg-card -i
**价值**: 语言理解→视觉传播

### 分类四：系统连接层

#### 12. ljg-skill-map - 技能地图
**目标**: 扫描所有已安装技能，渲染可视化总览
**用途**: 装库后的第一步自检工具

#### 13. ljg-x-download - X媒体下载
**依赖**: yt-dlp
**功能**: 下载X/Twitter帖子中的图片和视频

#### 14. ljg-travel - 旅行研究
**输出**: 深度文化研究文档 + 便携卡片
**侧重**: 历史分层、博物馆、古建、考古、人文脉络

#### 15. ljg-relationship - 关系分析
**方法**: 五层结构诊断 + 精神分析
**目标**: 帮用户"看见"关系真实结构

#### 16. ljg-roundtable - 圆桌讨论
**特点**: 求真导向的结构化多人辩证对话
**输出**: 每轮生成ASCII思考框架图

## Skill 文件结构标准

```markdown
name: skill-name
description: 一句话描述 + 触发词
user_invocable: true/false
version: x.x.x

# 核心目标
...

# 格式约束
- Org-mode/Markdown 语法
- ASCII Art 规范
- 文件命名规范
- 输出目录

# 红线（必须过）
1. ...
2. ...

# 工具箱（选用）
- ...

# 执行步骤
1. 获取内容
2. 处理
3. 输出

# 验收标准
- ...
```

## 关键洞察

### 1. 红线设计法
> "别只写目标，要写红线。让人知道'不要什么'比'要什么'更有约束力。"

### 2. 风格即工作流
> "`ljg-writes` 证明 skill 可以围绕'声音'设计，不只是流程。"

### 3. 从理解到传播
> "不是停在'想明白'，而是顺手做成交付物（card、flow）。"

### 4. 隐喻的力量
> "很多理解卡住，不是因为知识不够，而是因为隐喻不对。"

## 适用场景

**适合**:
- 内容创作者（读→消化→改写→卡片→发布）
- 知识工作者（论文、概念拆解、方法论整理）
- 想研究 skill 设计的人（最佳样板）

**不适合**:
- 只追求功能数量的人
- 希望完全中立、模板化输出的人

## 对我（小乔）的启发

1. **红线思维可以应用到我的回复风格** - 明确"不要什么"（AI腔、套话、过度礼貌）

2. **Skill 设计应该围绕"声音"而非"功能"** - 我的声音是什么？毒舌但有分寸、直接但有温度

3. **从理解到传播的完整链路** - 不只是回答问题，而是帮国栋形成可复用的认知资产

4. **隐喻的重要性** - 用更生动的隐喻替代抽象概念
