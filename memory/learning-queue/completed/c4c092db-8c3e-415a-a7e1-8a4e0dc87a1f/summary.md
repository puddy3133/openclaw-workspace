## 核心内容

### Superpowers 插件
- **GitHub**: https://github.com/obra/superpowers
- **Star数**: 11万
- **Anthropic插件市场**: 23万安装量，排名第2
- **本质**: 14个Skills组成的工作流系统，指导Agent如何完成任务

### 核心问题
大多数Agent拿到任务就开始写代码，会跳过设计、测试、review，产出不可维护的代码。

### Superpowers工作流（规划-拆解-执行-审查-复盘）

| 阶段 | Skill | 作用 |
|------|-------|------|
| 规划 | brainstorming | 苏格拉底式提问，一次一个问题，深入挖掘需求 |
| 隔离 | using-git-worktrees | 创建隔离工作区，分支开发 |
| 拆解 | writing-plans | 将设计文档拆成2-5分钟可完成的小任务 |
| 执行 | subagent-driven-development | 子Agent并行开发 |
| 审查1 | 独立审查Agent | 检查是否按需求实现 |
| 审查2 | 代码质量审查 | 检查代码规范、可维护性 |
| 终审 | requesting-code-review | 全局审查，模块集成检查 |
| 验证 | 最终验证 | 测试通过，合并回主分支 |

### 关键洞察
1. **规划2小时，执行10分钟，审查1小时** —— 前期规划比执行更重要
2. **苏格拉底式提问** —— 一次只问一个问题，根据回答继续追问
3. **小任务拆解** —— 2-5分钟可完成，每完成一个就验证
4. **能力一般的模型反而受益更大** —— 结构化流程弥补模型能力

### 适用场景
不仅限于开发，营销方案、PPT、数据分析等创造类任务都适用。

---

## 提取的经验

1. **需求挖掘**: 用苏格拉底式提问，一次一个问题，深入理解真实需求
2. **版本隔离**: 使用git worktree创建隔离工作区，避免污染主分支
3. **任务拆解**: 将大任务拆成2-5分钟可完成的小任务，便于验证
4. **分层审查**: 局部审查（功能实现）+ 全局审查（模块集成）
5. **子Agent协作**: 多个子Agent并行处理不同任务
