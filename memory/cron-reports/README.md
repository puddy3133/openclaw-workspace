# 定时任务执行规范

> 2026-03-04 确立

## 核心原则
**执行定时任务时，必须创建新会话，禁止占用主会话。**

## 消息推送规则
- **推送渠道**: 飞书消息（Feishu）
- **禁止**: TUI 或 Web 推送定时任务通知
- **方式**: 通过 `message` 工具发送到用户飞书
- **格式**: 简洁明了，包含任务名称和执行结果摘要

## 执行流程

```
定时触发
    ↓
创建新会话 (isolated)
    ↓
执行任务
    ↓
生成报告
    ↓
结束新会话
    ↓
恢复最近主会话（如有）
```

## 会话管理

### 1. 创建新会话
```javascript
sessions_spawn({
  agentId: "main",
  task: "执行定时任务: {task-name}",
  runTimeoutSeconds: 300,
  cleanup: "delete"  // 完成后自动清理
})
```

### 2. 报告生成
**报告位置**: `memory/cron-reports/{job-id}-{YYYY-MM-DD}.md`

**报告内容**:
```markdown
# 定时任务执行报告

## 任务信息
- **任务ID**: {job-id}
- **任务名称**: {name}
- **执行时间**: YYYY-MM-DD HH:MM:SS (Asia/Shanghai)
- **执行时长**: X 秒

## 执行结果
- **状态**: ✅ 成功 / ❌ 失败
- **输出摘要**: 
- **错误信息**: （如有）

## 关键操作
（记录执行的关键步骤）

## 建议
（如有问题，给出建议）
```

### 3. 恢复主会话
```
定时任务完成
    ↓
检查是否有活跃主会话
    ↓
如有 → 发送恢复通知到主会话
如无 → 静默结束
```

## 禁止行为

| 禁止 | 原因 |
|------|------|
| 在主会话执行定时任务 | 打断用户当前工作 |
| 不生成执行报告 | 无法追溯和审计 |
| 失败不通知 | 用户无法及时知晓 |
| 占用会话不释放 | 资源浪费 |

## 配置示例

```json
{
  "id": "example-task",
  "name": "示例定时任务",
  "enabled": true,
  "schedule": {
    "kind": "every",
    "everyMs": 86400000,
    "anchorMs": 1772678400000
  },
  "sessionTarget": "isolated",  // ✅ 必须
  "payload": {
    "kind": "agentTurn",
    "message": "执行任务...",
    "timeoutSeconds": 300
  },
  "delivery": {
    "mode": "announce"  // 完成后通知主会话
  }
}
```
