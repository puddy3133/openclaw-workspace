# Recurring Tasks - 周期性任务定义

> Heartbeat 每30分钟检查是否到期

## 当前周期任务

### Heartbeat 巡检
- **频率**：每30分钟
- **动作**：
  1. 扫 tasks/active/ → 检查TTL
  2. 扫 tasks/waiting/ → 检查是否可解阻塞
  3. 扫 tasks/recurring/ → 检查是否到期
  4. 生成巡检报告（不自动执行，只汇报）
- **下次执行**：启动后30分钟

### 深夜归档
- **频率**：每天 02:00 EST
- **动作**：
  1. 整理今日日志 → 生成明日待办
  2. 归档已完成任务
  3. 更新 memory/INDEX.md
  4. 生成明日简报（可选发送）
- **约束**：
  - 不执行对外发送消息
  - 不执行涉及金钱/删除的操作
  - 不执行需要用户确认的操作

### 周回顾生成
- **频率**：每周日 22:00 EST
- **动作**：生成本周回顾到 memory/weekly/
- **下次执行**：2026-02-28 22:00 EST

---

*添加新周期任务时，记得设置明确的触发条件和动作*
