# 第三章 系统架构设计

## 一、总体架构设计

### 1.1 架构设计原则

本系统架构设计遵循以下原则：

**高可用性原则**：系统采用微服务架构，各服务独立部署，单点故障不影响整体运行，确保系统7×24小时稳定运行。

**高扩展性原则**：采用容器化部署，支持水平扩展，可根据业务负载动态调整资源，满足未来3-5年业务发展需求。

**高安全性原则**：遵循等保二级安全要求，从网络、主机、应用、数据四个层面构建安全防护体系。

**标准化原则**：遵循教育部、人社部相关标准规范，确保数据格式、接口规范符合行业标准。

### 3.2 数据流向设计

**业务数据流向**：
1. 业务系统产生数据 → 写入MySQL业务库
2. 通过CDC（Change Data Capture）实时同步至数据仓库
3. 数据仓库进行清洗、转换、聚合
4. 生成报表、大屏、分析所需数据

**外部数据流向**：
1. 通过ETL工具从教务、科研、学工等系统抽取数据
2. 数据质量校验后进入数据仓库
3. 与业务数据融合，形成完整教师画像

**功能描述**：提供智能问答、业务办理引导、数据查询等服务。

**技术实现**：
- 基于本地部署的大语言模型（如ChatGLM3-6B）
- RAG（检索增强生成）技术，结合人事知识库

- 访问控制：根据用户角色限制AI回答范围

**交互流程**：
```
用户提问 → 意图识别 → 知识库检索 → 大模型生成 → 安全过滤 → 返回结果
```

# 第六章 接口设计

## 一、接口设计规范

### 1.1 RESTful API规范

**URL设计**：
```
https://api.hr.szpt.edu.cn/{version}/{module}/{resource}/{action}
```

| 组成部分 | 说明 | 示例 |
|----------|------|------|
| version | API版本 | v1, v2 |
| module | 模块名称 | workbench, recruit, employee |
| resource | 资源名称 | person, contract, leave |
| action | 操作（可选） | export, import, approve |

**HTTP方法**：
| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 查询资源 | GET /v1/employee/person/1001 |
| POST | 创建资源 | POST /v1/employee/person |
| PUT | 更新资源 | PUT /v1/employee/person/1001 |
| DELETE | 删除资源 | DELETE /v1/employee/person/1001 |
| PATCH | 部分更新 | PATCH /v1/employee/person/1001/status |

**响应格式**：
```json
{
    "code": 200,
    "message": "操作成功",
    "data": {},
    "timestamp": 1704067200000,
    "requestId": "req_123456789"
}
```

**状态码**：
| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |

## 三、数据安全设计

### 3.1 数据分类分级

**数据分类**：
| 分类 | 数据类型 | 保护级别 |
|------|----------|----------|
| 公开数据 | 部门名称、岗位名称 | 公开 |

| 绝密 | 国密算法 | HTTPS | 最小授权 | 实时审计 |

### 3.2 数据加密方案

**存储加密**：
```java
// 敏感字段加密示例
@Column(name = "id_card")
@Convert(converter = AesEncryptConverter.class)
private String idCard;

// 加密转换器
public class AesEncryptConverter implements AttributeConverter<String, String> {
    @Override
    public String convertToDatabaseColumn(String attribute) {
        return AesUtil.encrypt(attribute, secretKey);
    }
    
    @Override
    public String convertToEntityAttribute(String dbData) {
        return AesUtil.decrypt(dbData, secretKey);
    }
}
```

**传输加密**：
- 强制HTTPS协议
- TLS 1.2+版本
- 证书双向认证（敏感接口）
