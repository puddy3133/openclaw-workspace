# 从零到一：用 OpenClaw + Fish.audio 打造专属 AI 播客

> **项目亮点**：实现双 AI 角色（大乔×小乔）语音对话播客，探索 TTS 技术在 AI 智能体中的应用边界
> 
> **核心关键词**：OpenClaw、Fish.audio、TTS、AI 播客、自定义音色、Edge TTS 降级方案

---

## 一、项目背景与灵感来源

### 1.1 为什么做这个项目？

在 WaytoAGI OpenClaw 小龙虾训练营的第 7 课《赋予声带——让小龙虾开口说话》中，我们接触到了 TTS（文本转语音）技术。但课程示例较为基础，我们想挑战一个更有意思的场景：

**让两个 AI 助手（大乔和小乔）进行对话，生成一段完整的播客节目。**

这涉及几个技术难点：
- 如何为不同 AI 角色分配不同音色？
- 如何实现自然的对话节奏（停顿、语气）？
- 如何处理长文本的多轮对话合成？
- 当主方案失败时，如何快速降级？

### 1.2 角色设定

| 角色 | 定位 | 音色风格 | 部署位置 |
|:---|:---|:---|:---|
| **大乔** | 云端 AI 助手 | 温柔知性 | 腾讯云服务器 |
| **小乔** | 本地 AI 助手 | 活泼灵动 | 本地 Mac |

## 二、技术方案选型与踩坑

### 2.1 第一方案：Fish.audio（理想方案）

**选择理由**：
- 支持**声音复刻**，5 秒录音即可克隆音色
- 中文语音质量极佳，情感丰富
- API 简单易用

**踩坑实录**：

```bash
# 错误信息
{"message":"Insufficient Balance","status":402}
```

**问题分析**：
- Fish.audio 新用户虽有免费额度（约 10000 字符），但极易用完
- 额度用完后，API 直接返回 402 错误，没有任何预警
- 充值流程不够便捷

**经验总结**：
> 💡 **教训**：在生产环境中使用 TTS 服务时，务必提前测试额度消耗速度，并准备备选方案。

---

### 2.2 降级方案：Edge TTS（免费稳定）

当 Fish.audio 额度耗尽后，我们迅速切换到 **Edge TTS**（微软 Edge 浏览器的语音合成引擎）。

**优势**：
- ✅ 完全免费，无额度限制
- ✅ 无需 API Key
- ✅ 支持中文，语音质量较好
- ✅ 支持多种预设音色

**局限性**：
- ❌ 不支持自定义音色（无法克隆大乔/小乔的声音）
- ❌ 只能选择预设语音

**音色映射方案**：

| 角色 | 原计划音色 | Edge TTS 替代 | 相似度 |
|:---|:---|:---|:---|
| 大乔 | Fish.audio 克隆 | `zh-CN-XiaoyiNeural`（温柔女声） | 70% |
| 小乔 | Fish.audio 克隆 | `zh-CN-XiaoxiaoNeural`（活泼女声） | 75% |

---

### 2.3 备选方案调研：火山引擎豆包

在解决问题的过程中，我们还调研了**火山引擎豆包语音合成**：

**亮点**：
- 支持**声音复刻大模型**，5 秒定制专属音色
- 新用户有免费额度
- 价格较低（22.5元/10万字）

**适用场景**：
- 需要自定义音色但 Fish.audio 额度不足时
- 企业级应用，需要稳定服务

## 四、踩坑经验全记录

### 4.1 坑点一：TTS 服务额度陷阱

**现象**：Fish.audio 突然返回 402 错误

**解决方案**：
1. 提前测试额度消耗速度
2. 准备降级方案（Edge TTS）
3. 监控额度使用情况

```python
# 额度检查伪代码
def check_tts_balance():
    balance = api.get_balance()
    if balance < threshold:
        alert("TTS 额度不足，请充值或切换降级方案")
        switch_to_backup_tts()
```

### 4.2 坑点二：音频合并失败

**现象**：多个 MP3 片段合并后时长不对

**原因**：不同片段的采样率不一致

**解决方案**：
```bash
# 统一转换为相同格式
ffmpeg -i input.mp3 -ar 24000 -ac 1 -acodec libmp3lame output.mp3
```

### 4.3 坑点三：长文本截断

**现象**：长段落合成后被截断

**解决方案**：
- 将长文本拆分为多个短句
- 每句控制在 200 字以内
- 分别合成后合并

### 4.4 坑点四：中文标点问题

**现象**：某些标点符号导致语音合成异常

**解决方案**：
```python
# 清洗文本
def clean_text(text):
    # 替换特殊标点
    text = text.replace('"', '')
    text = text.replace("'", '')
    # 统一中文标点
    text = text.replace('...', '…')
    return text
```

## 六、给其他开发者的建议

### 6.1 技术选型建议

| 需求场景 | 推荐方案 | 理由 |
|:---|:---|:---|
| 需要自定义音色 | Fish.audio / 火山引擎豆包 | 支持声音复刻 |
| 免费、无额度限制 | Edge TTS | 完全免费，质量可接受 |
| 企业级应用 | 火山引擎豆包 | 稳定，价格可控 |
| 快速原型验证 | Edge TTS | 无需注册，立即可用 |

### 6.2 开发流程建议

1. **先跑通主流程**：用 Edge TTS 快速验证可行性
2. **再优化音色**：接入 Fish.audio 提升体验
3. **最后加监控**：确保服务稳定性

### 6.3 成本控制建议

```
开发阶段：Edge TTS（免费）
    ↓
测试阶段：Fish.audio 免费额度
    ↓
生产阶段：根据用量选择 Fish.audio / 火山引擎
```

## 八、总结

本次项目最大的收获是：**在技术选型时，务必准备 Plan B**。

当我们满怀信心地使用 Fish.audio 时，额度问题给了我们当头一棒。但正是这个"坑"，让我们深入了解了 Edge TTS、火山引擎等多个方案，最终不仅解决了问题，还形成了一套完整的 TTS 技术选型方法论。

**核心经验**：
1. 免费方案（Edge TTS）是快速验证的最佳选择
2. 付费方案（Fish.audio/火山引擎）是生产环境的保障
3. 降级方案是系统稳定性的关键
