# 启动恢复流程

> 每次启动执行，目标：5秒内完成恢复

## 执行顺序

### 1. 读取身份信息（必须）

```bash
read SOUL.md       # 我是谁
read USER.md       # 我在帮谁
```

### 2. 读取当前状态（必须）

```bash
read .state/current.md        # 我之前在干嘛
read .state/focus.md          # 当前专注任务
```

### 3. 读取记忆索引（必须）

```bash
read memory/INDEX.md          # 快速定位关键信息
```

### 4. 读取最近日志（必须）

```bash
read memory/2026-02-22.md     # 今天发生了什么
read memory/2026-02-21.md     # 昨天发生了什么（如果今天没有记录）
```

### 5. 扫描当前任务（必须）

```bash
ls tasks/active/              # 列出当前任务
```

### 6. 生成启动摘要

```markdown
我回来了，当前状态是：

**正在处理**：{current.md 中的任务}
**进度**：{current.md 中的进度}
**当前任务**：{focus.md 中的任务}
**活跃任务数**：{active/ 目录中的文件数}

上次我们在做 {任务名称}，进度是 {进度}，接下来要 {下一步行动}。
```

## 示例启动摘要

```
我回来了，当前状态是：

正在处理：配置永续Agent架构 v2.0
进度：基础架构已创建，待测试
当前任务：TASK-001 永续Agent架构配置
活跃任务数：1

上次我们在配置永续Agent架构，已经创建了三层架构的基础文件，接下来要编写启动恢复流程脚本和配置Heartbeat巡检。
```

## 注意事项

1. **分层读取**：启动时只读必要文件，细节按需加载
2. **快速恢复**：目标5秒内完成，用户感知不到延迟
3. **状态确认**：启动后确认所有关键文件存在，缺失则重建
4. **上下文连续**：确保"我当时在想什么"能够被恢复

## 文件依赖关系

```
SOUL.md ──┐
USER.md ──┼──> 身份确认
          │
.state/current.md ──┐
.state/focus.md ────┼──> 状态恢复
                    │
memory/INDEX.md ────┼──> 记忆定位
memory/YYYY-MM-DD.md┤
                    │
tasks/active/* ─────┴──> 任务列表
```
